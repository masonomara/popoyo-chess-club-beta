## Task

Write a complete Postgres schema to `docs/schema.sql` that covers everything in `docs/01-goal.md`. This is the entire architecture step — queries, server actions, and components all derive from it.

**Tables**

- One table per entity, no exceptions
- Every column: explicit type + `NOT NULL` unless null is meaningful
- `UNIQUE` constraints wherever the data model requires it
- UUID primary keys: `DEFAULT gen_random_uuid()`
- `created_at TIMESTAMPTZ NOT NULL DEFAULT now()` on every table

**Foreign keys**

- Explicit `FOREIGN KEY` constraint on every relationship
- `ON DELETE CASCADE` by default — use `SET NULL` or `RESTRICT` only with a comment explaining why

**Row Level Security**

- `ALTER TABLE [table] ENABLE ROW LEVEL SECURITY;` on every table
- Explicit `CREATE POLICY` for SELECT, INSERT, UPDATE, DELETE per table
- Policies must match the roles in `docs/01-goal.md` — no permissive catch-alls; if a role can't do it, no policy exists for it

**Postgres functions**

- `CREATE OR REPLACE FUNCTION` for any mutation touching more than one table or requiring atomicity
- Named clearly — these are called as RPCs from server actions

**Indexes**

- Index every foreign key column
- Index columns that appear in frequent `WHERE` or `ORDER BY` clauses per the flows in `docs/01-goal.md`

Begin the file with:

```sql
-- Schema for [project name]. Generated by /schema.
-- Run in the Supabase SQL Editor before seed.sql.
```

After writing the file, output a table: each table name, column count, and one-line description.

<details>
<summary>Why this approach</summary>

`app/types/database.ts` is generated from this schema and is the single source of truth for all types in the codebase. No type is defined anywhere else. If code needs a cast, the schema is wrong — fix the schema, not the code.

Types are executable contracts, not documentation. Schema-first means: define the data model once, generate everything from it, and never let application code drift from what the database actually holds.

To generate types after running the schema:

1. Confirm `SUPABASE_PROJECT_ID` is set in `.env.local` (the project ref from the Supabase dashboard URL)
2. Run `npm run types`
3. Confirm `app/types/database.ts` now contains the generated types

`types` script for `package.json`:

```json
"types": "npx --yes supabase gen types typescript --project-id $SUPABASE_PROJECT_ID > app/types/database.ts"
```
